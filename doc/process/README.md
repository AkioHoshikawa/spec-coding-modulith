# Process

## 概要

設計・開発を行うにあたりどのように進めれば良いかを説明した人間・AIの両者向け手順です。

このProcessを用いる目的は以下です。

- タスクを遂行するときの手順を各作業者で統一させる
- タスクごとのレビュー観点を明確にし、レビューの効率を向上させる

Processには以下の要素が含まれます。

- Target: Processの目標
- Prompt Example: 人間がCoding AgentにProcessを依頼する場合のPrompt例
- Phase: 作業内容をStepごとに区切ったもの
    - Input, Process, Output, Checklistの4つで構成される

## 使い方

- AIに作業を依頼する場合は各Processに対応するAgentがあるため、それを利用する。
- 手動で対応する場合はProcessの内容に従って作業を進める。
