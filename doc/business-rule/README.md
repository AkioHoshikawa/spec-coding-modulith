# ビジネスルール

- 定義: ビジネスルールは、業務上の制約や判断基準をコードや仕様に落とし込んだものです。たとえば在庫の引当条件、プロモーションの適用順序、返品可否の判定などが該当します。
- 目的: 業務の一貫性と正しさを保つこと。ルールを明確にすると、実装・テスト・レビューがしやすくなります。
- どのように書くか（開発者向けの直感）:
	- 条件（いつ）: どんな入力・状態のときにルールが働くか
	- 振る舞い（どうする）: 期待される出力や状態遷移
	- 例外（いつ適用しない）: 例外ケースや優先順位

- テスト: ルールは必ずテストで表現しましょう。代表的な正常系、境界、例外ケースを用意します。
- 変更時の注意: ビジネスルールの変更は業務影響が大きいので、変更履歴と理由を残し、関係者の承認を得てからリリースします。
